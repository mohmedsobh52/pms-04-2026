
## نظرة عامة

المواصفات (36 قسم) ضخمة جداً وتفوق حجم دورة واحدة. سأنفذها على **8 مراحل متتالية**، كل مرحلة قابلة للاختبار والاستخدام مستقلة. جميع الأعمال ستُبنى على المكونات الموجودة (`CostAnalysisPage`, `SmartCostEnginePanel`, `ProjectInfoBar`, `CostKpiGrid`, `ItemDetailsDrawer`, `SensitivityScenarios`, `CostVersionsPanel`, `AiCostAdvisorPanel`) **دون إعادة بناء** أو تغيير الهوية البصرية (Teal Trust + Tajawal/Cairo + RTL).

## المراحل

### المرحلة 1 — تعزيز شريط المشروع + المؤشرات (الأقسام 3، 4، 6)
- توسيع `ProjectInfoBar`: حالة التحليل (مسودة/مراجعة/معتمد/مرفوض/مؤرشف)، رقم الإصدار، آخر مستخدم، أزرار "تعديل"/"نسخة جديدة".
- توسيع `CostKpiGrid` إلى 18 بطاقة (قبل/بعد الضريبة، عدد المكتمل/الناقص/يحتاج مراجعة، متوسط الثقة، التوفير المتوقع، نسبة الاكتمال).
- Tooltips شرح لكل مؤشر (كيفية الحساب + الحد المقبول + الإجراء).

### المرحلة 2 — تطوير محرك التحليل الذكي (القسم 5)
- توسيع `SmartCostEnginePanel`: قائمة تشغيل (الحالي/المحدد/الكل/المعدل/الناقص/إعادة كامل).
- Progress Bar حقيقي، حالات (7 حالات)، أزرار (إيقاف/إعادة/تفاصيل خطأ/تنزيل سجل).
- عرض: آخر تحليل، المستخدم المنفذ، عدد المشكلات، التوفير المتوقع.

### المرحلة 3 — البحث والفلاتر المتقدمة (القسم 7)
- توسيع `CostItemsToolbar`: 17 فلتر (تصنيف/وحدة/حالة/ثقة/اقتراحات/تحذيرات/اعتماد/مستخدم/تاريخ/نطاق تكلفة/نسب/مصدر سعر).
- 12 خيار ترتيب. زر مسح شامل + عداد نتائج.

### المرحلة 4 — تطوير الجدول (الأقسام 8، 9، 10)
- 26 عمود مع إظهار/إخفاء + حفظ إعدادات المستخدم في localStorage.
- Inline editing مع مؤشرات (جاري/تم/فشل).
- إجراءات جماعية موسعة (نسخ/حذف/أرشفة/تحليل/تصدير/تغيير تصنيف/اعتماد).
- Drag-and-drop لإعادة الترتيب، تثبيت أعمدة، pagination (25/50/100/200).
- نافذة "إضافة بند" مع خيارات (حفظ / حفظ وإضافة / حفظ وتحليل / حفظ كقالب).

### المرحلة 5 — نافذة تفاصيل البند + الافتراضات + العملات (الأقسام 11، 12، 13)
- توسيع `ItemDetailsDrawer` إلى 8 تبويبات (أساسي/تفصيل/مواد/عمالة/معدات/افتراضات/اقتراحات/سجل/مرفقات).
- قسم "افتراضات المشروع" مع حساب "البنود المتأثرة" قبل التطبيق.
- إدارة عملات متعددة + ضرائب لكل بند + إعفاءات.

### المرحلة 6 — الاقتراحات + كشف الشذوذ + الحساسية + السيناريوهات (الأقسام 14، 15، 16، 17، 18)
- توسيع `AiCostAdvisorPanel`: تصنيف 10 أنواع اقتراحات، إجراءات (تطبيق/تعديل/تجاهل بسبب/تأجيل/تعليق/تطبيق على مشابهة/Undo).
- كاشف شذوذ (15 حالة).
- توسيع `SensitivityScenarios` بمتغيرات إضافية + مقارنة حتى 4 سيناريوهات.
- تطوير الرسم الدائري (11 فئة، تفاعلي، مقارنة قبل/بعد) + رسوم إضافية.

### المرحلة 7 — الأسعار/الموردين + القوالب + الاستيراد/التصدير (الأقسام 19، 20، 21، 22)
- ربط عروض أسعار متعددة بالمادة + مقارنة/اختيار معتمد.
- إدارة قوالب (إنشاء/نسخ/أرشفة/مشاركة).
- استيراد Excel/CSV مع معاينة ومطابقة أعمدة وتراجع.
- تصدير 6 أنواع تقارير (مختصر/تفصيلي/تنفيذي/فني/مراجعة/فروقات).

### المرحلة 8 — Workflow + التعليقات + الإشعارات + الإصدارات + الصلاحيات (الأقسام 23–27)
- Workflow اعتماد (7 حالات + 6 أدوار) عبر جداول `workflow_*` الموجودة.
- تعليقات (على بند/عام/mentions/رد/حل) عبر `analysis_comments`.
- إشعارات عبر `notifications` (نظام v2 الموجود).
- Version control عبر `CostVersionsPanel` + snapshots تلقائية.
- RBAC عبر `useUserRoles` — إخفاء/تعطيل الأزرار.

## قواعد عبر جميع المراحل
- **حساب مركزي**: كل الحسابات تمر عبر `src/lib/cost-engine/formulas.ts` + `derive-totals.ts` (موجود). لا تكرار.
- **مصدر بيانات واحد**: تعديل بند → تحديث فوري لكل بطاقة/رسم/إجمالي.
- **بدون بيانات وهمية**: كل شيء من قاعدة البيانات.
- **RTL + Tajawal/Cairo + Teal Trust**: لا تغييرات على الهوية.
- **الأداء**: Virtualization للجدول (موجود عبر `DataTable`)، debounce للبحث، React Query cache.
- **Accessibility**: Keyboard nav، ARIA، تباين، Focus واضح.
